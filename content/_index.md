# Schemion Docs

Добро пожаловать в документацию Schemion. Здесь собраны материалы по архитектуре, API, формату датасетов, развертыванию и разработке платформы.

## С чего начать

- [Документация](/docs/) - карта всех технических разделов.
- [Архитектура](/docs/architecture/) - как связаны API, воркеры, брокер, база и MinIO.
- [API](/docs/api/) - основные эндпоинты, форматы запросов и ограничения.
- [Формат датасета](/docs/data/dataset-format/) - структура ZIP-архива и YOLO-разметки.

## Типовой путь

1. Поднять окружение через Docker Compose.
2. Зарегистрировать пользователя и получить JWT.
3. Загрузить датасет и модель.
4. Создать задачу обучения или инференса.
5. Отслеживать статус задачи через REST или SSE.

Schemion строится вокруг простого контура: API принимает запросы, Bobber доставляет задачи воркерам, PostgreSQL хранит метаданные, а MinIO держит все тяжелые артефакты.
